AI 채팅 시스템 구조 설명서

사용자가 "삼성전자 시세 알려줘"라고 입력하면 어떤 일이 일어날까요?
시스템의 모든 원리를 처음부터 끝까지 쉽게 설명합니다.

FastAPI (Python) Ollama llama3.1:8b Oracle DB MongoDB Spring Boot API

🗺️ 시스템 개요

이 채팅 시스템은 주식 투자 앱을 위한 AI 어시스턴트입니다. 단순한 시세 조회부터 복잡한 포트폴리오 분석까지 자연어로 처리합니다.

💬 대화 인터페이스

사용자가 한국어로 질문하면 AI가 이해하고 적절한 금융 데이터를 찾아 답변합니다.

POST /chat

🧠 2가지 처리 방식

간단한 조회는 규칙 기반 템플릿으로 빠르게, 복잡한 질문은 AI 에이전트가 단계별로 추론합니다.

📊 다양한 데이터 소스

실시간 주가(Spring API), 뉴스(MongoDB), 계좌/거래 내역(Oracle DB) 를 통합해 답변합니다.

💡
핵심 설계 원칙: 전체 질문의 약 80%는 AI 없이 규칙 기반으로 처리하여 빠르고 정확하게 응답합니다. 나머지 20%의 복잡한 질문만 AI 언어 모델이 처리합니다. 이 덕분에 응답 속도와 정확도를 모두 확보합니다.

🔄 메시지 처리 흐름

"삼성전자 시세 알려줘"를 입력했을 때 시스템 내부에서 일어나는 10단계 과정입니다.

1

사용자 입력 수신

앱에서 POST /chat 요청이 들어옵니다. 본문에는 메시지, 헤더에는 Authorization: Bearer <JWT토큰>이 포함됩니다.

2

인증 확인 (auth.py)

JWT 토큰을 Spring API /api/users/me로 검증합니다. 같은 토큰은 30분간 캐시하여 중복 요청을 줄입니다. 검증 실패 시 401 오류를 반환합니다.

3

입력 유효성 검사 (main.py)

불완전한 한글 자모음(ㅅㅅ), 1~2자 티커만 입력 등 의미없는 메시지를 걸러냅니다. 통과하면 다음 단계로 진행합니다.

4

빠른 경로 감지 — 종목 단축키 (main.py)

직전 대화에서 시세나 뉴스를 조회한 상태에서 종목명만 입력하면 ("SK하이닉스" 등), 즉시 같은 조회를 반복합니다. 의도 분류 과정을 건너뛰어 응답이 빠릅니다.

5

의도(Intent) 분류 (chatbot/router.py)

정규식 패턴 15개 이상으로 사용자의 의도를 파악합니다. 예: "코스피" → index, "시세" → chart_price, "뉴스" → stock_news. 우선순위 순서대로 매칭합니다.

6

의도 보정 (main.py 문맥 검사)

감지된 의도가 문맥상 맞는지 확인합니다. "그 종목", "아까" 같은 대명사, 여러 종목 비교, 종목코드 누락 등이면 AI 에이전트로 전환합니다.

7

핸들러 디스패치 (chatbot/dispatcher.py)

의도에 따라 두 경로 중 하나로 처리됩니다: 템플릿 경로(빠른 API 조회 → 포맷 출력) 또는 에이전트 경로(AI가 추론하며 처리).

8

[에이전트 경로만] AI 추론 (agent/llm_agent.py)

Ollama llama3.1:8b 모델이 ReAct 방식으로 5개의 도구(시세조회, 뉴스조회, 포트폴리오, 시황, 거래내역)를 최대 5번 호출하며 답을 만들어냅니다.

9

대화 기록 저장 (db/mongo.py)

이번 대화 턴(사용자 발언 → 도구 호출 → 도구 결과 → AI 답변)을 MongoDB에 저장합니다. 다음 질문 시 최근 6턴을 불러와 문맥으로 활용합니다.

10

최종 응답 반환

앱으로 JSON 응답을 돌려줍니다. type이 "order"이면 프론트엔드가 매수/매도 다이얼로그를 표시하고, "text"이면 일반 채팅 메시지로 보여줍니다.

⚡ 두 가지 처리 경로

의도가 파악되면 복잡도에 따라 두 경로 중 하나로 처리됩니다.

⚡ 템플릿 경로 (약 80%)

규칙 기반으로 즉시 API를 호출하고 미리 정의된 형식으로 응답합니다. 빠르고 일관성 있습니다.

  • 📈 지수 조회 (코스피, 나스닥)
  • 💱 환율 조회
  • 🏆 상승/하락 랭킹
  • 📊 종목 시세 (단일 종목)
  • 💰 잔고 / 예수금
  • 📰 종목 뉴스 (단일 종목)
  • 🌏 시황 요약 (한국/미국)
  • 🛒 매수/매도 주문 (UI 팝업)
  • 💱 환전 주문 (UI 팝업)

🧠 AI 에이전트 경로 (약 20%)

LLM이 여러 도구를 순차적으로 호출하며 복잡한 질문을 단계별로 추론합니다.

  • 🔍 복수 종목 비교
  • 📋 포트폴리오 심화 분석
  • 🔗 대명사 활용 질문 ("그 종목...")
  • 📅 복잡한 거래 내역 질문
  • 🔢 조합 질문 (포트폴리오 + 시세)
  • ❓ 분류 불가 자유 질문
📝
왜 두 경로로 나누나요? AI 에이전트는 유연하지만 응답 시간이 1~3초로 느립니다. 반면 템플릿 경로는 100ms 이내로 답변합니다. "코스피 얼마야?" 같은 간단한 질문에 AI를 쓰는 건 비효율적이기 때문에, 복잡도에 따라 적절히 선택합니다.

🤖 AI 에이전트 동작 원리

llm_agent.py의 ReAct 방식 — Reason(추론) + Act(행동)을 반복하며 답을 찾아갑니다.

ReAct 반복 사이클

1

질문 분석

MongoDB에서 최근 6턴 불러옴 + 시스템 프롬프트와 함께 Ollama에 전송

2

도구 선택 및 호출

5개 도구 중 필요한 것을 선택해 실제 데이터 조회

3

결과 통합 및 재추론

도구 결과를 받아 추가 도구 필요 여부 판단 (최대 5회 반복)

4

최종 답변 생성

모든 정보를 종합해 자연어 답변 반환

5가지 도구

get_stock_price 시세

종목코드 → 현재가, 등락률, 거래량

get_stock_news 뉴스

종목 관련 최신 뉴스 3건 (MongoDB)

get_portfolio_info 포트폴리오

보유 종목, 수익률, 섹터, MDD 등 (Oracle)

get_market_summary 시황

한국/미국 시장 전체 분위기 요약

get_trade_history 거래

매수/매도 기록 조회 (Oracle)

환각 방지 (Hallucination Prevention)

AI가 숫자를 지어내지 않도록 여러 방어 장치가 적용되어 있습니다.

숫자 문자열 변환

도구 결과를 AI에 전달할 때 원시 숫자(922000) 대신 형식화된 문자열("922,000원")로 변환합니다. AI가 숫자를 재계산할 여지를 없앱니다.

온도(Temperature) 0

창의성 없이 최대한 결정론적으로 동작합니다. 동일한 입력에 동일한 출력 → 예측 가능하고 안전합니다.

오류 즉시 중단

도구 호출 중 오류 발생 시 AI가 답을 추측하지 않고 즉시 "데이터를 가져올 수 없습니다"를 반환합니다.

4개의 데이터 소스가 역할을 나눠 담당합니다.

대화 기록이 저장되는 방식

MongoDB에는 한 번의 대화 교환(턴)이 아래와 같이 하나의 문서로 저장됩니다.

// MongoDB 한 턴 저장 예시 { "account_id": "1", "timestamp": ISODate("2024-03-31T10:30:00Z"), "messages": [ { "role": "user", "content": "삼성전자 시세" }, { "role": "assistant", "tool_calls": ["get_stock_price(005930)"] }, { "role": "tool", "content": "현재가: 75,400원, +0.54%" }, { "role": "assistant", "content": "삼성전자 현재가는 75,400원입니다..." } ] }

다음 질문 시 이 기록 최근 6턴을 불러와 AI의 문맥으로 활용합니다. 로그아웃 후 재로그인 시 새 세션으로 초기화됩니다.

🧠 에이전트 경로
