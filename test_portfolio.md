# 포트폴리오 테스트 시나리오

> **사전 조건**
>
> - MongoDB chat_history 초기화 권장 (세션 오염 방지)
> - 보유 종목: 마이크론 테크놀로지, 삼성전자, 아마존닷컴, 엔비디아, 미래에셋증권 (테스트 계정 기준)
> - 결과: ✅ 통과 / ❌ 실패 / ⚠️ 부분 통과

---

## 1. 단순 포트폴리오 조회 (템플릿 경로)

> `_PORTFOLIO_SIMPLE_RE` 매칭 → `format_portfolio()` 템플릿 반환

| 1-3 | `포트폴리오 현황`     | 템플릿    | 위와 동일                                           |      |
| 1-4 | `포트폴리오 조회해줘` | 템플릿    | 위와 동일                                           |      |
| 1-5 | `포폴 보여줘`         | 템플릿    | 위와 동일                                           |      |

## 2. 지표별 분석 — agent 경로 + \_해설 활용

> `_detect_metric_type` 주석 처리 → 모두 agent 경로
> `get_portfolio_info` tool 결과에 `_해설` 필드 포함 → LLM이 자연어로 설명

### 2-A. 수익률 (returns)

| #   | 입력                                    | 기대 tool                               | 기대 응답 포인트                                            | 결과 |
| --- | --------------------------------------- | --------------------------------------- | ----------------------------------------------------------- | ---- |
| 2-1 | `내 수익률 어때`                        | `get_portfolio_info(returns)`           | 1·3·6개월 수익률 언급, best/worst 종목 언급                 |      |
| 2-2 | `포트폴리오 수익률 알려줘`              | `get_portfolio_info(returns)`           | 위와 동일                                                   |      |
| 2-3 | `6개월 수익률 어때`                     | `get_portfolio_info(returns)`           | 6개월 수치 포함                                             |      |
| 2-4 | `수익이 가장 높은 종목`                 | `get_portfolio_info(returns 또는 risk)` | best_stock 이름 + 수익률                                    |      |
| 2-5 | `손실이 가장 큰 종목`                   | `get_portfolio_info(risk)`              | worst_stock 이름 + 수익률                                   |      |
| 2-6 | `포트폴리오 내 제일 수익률이 좋은 종목` | `get_portfolio_info(returns 또는 risk)` | best_stock 답변 (이전에 "SK은 미보유" 오류 발생했던 케이스) |      |

### 2-B. 리스크 (risk)

| #    | 입력                        | 기대 tool                  | 기대 응답 포인트                                                                       | 결과 |
| ---- | --------------------------- | -------------------------- | -------------------------------------------------------------------------------------- | ---- |
| 2-7  | `리스크 지표 보여줘`        | `get_portfolio_info(risk)` | MDD·변동성·회복 필요 수익률 + 의미 설명                                                |      |
| 2-8  | `포트폴리오 내 리스크 분석` | `get_portfolio_info(risk)` | 각 수치 의미 풀어서 설명 (이전에 "조회한 데이터를 보여드렸어요." 오류 발생했던 케이스) |      |
| 2-9  | `MDD 얼마야`                | `get_portfolio_info(risk)` | MDD 수치 포함 답변                                                                     |      |
| 2-10 | `변동성 어때`               | `get_portfolio_info(risk)` | 변동성 수치 포함 답변                                                                  |      |
| 2-11 | `평가손익 얼마야`           | `get_portfolio_info(risk)` | unrealized_pnl 수치                                                                    |      |
| 2-12 | `실현손익 알려줘`           | `get_portfolio_info(risk)` | realized_pnl 수치                                                                      |      |

### 2-C. 섹터/구성 (sector)

| #    | 입력                      | 기대 tool                    | 기대 응답 포인트             | 결과 |
| ---- | ------------------------- | ---------------------------- | ---------------------------- | ---- |
| 2-13 | `섹터 비중 알려줘`        | `get_portfolio_info(sector)` | 섹터별 비중 + 국내/해외 비율 |      |
| 2-14 | `국내랑 해외 비중이 어때` | `get_portfolio_info(sector)` | domestic/foreign ratio       |      |
| 2-15 | `포트폴리오 구성 어때`    | `get_portfolio_info(sector)` | 종목별 비중 상위 언급        |      |
| 2-16 | `종목 비중 보여줘`        | `get_portfolio_info(sector)` | 섹터 구성                    |      |

### 2-E. 보유 종목 (holdings)

| #    | 입력                    | 기대 tool                      | 기대 응답 포인트                                    | 결과 |
| ---- | ----------------------- | ------------------------------ | --------------------------------------------------- | ---- |
| 2-20 | `보유 종목 뭐야`        | `get_portfolio_info(holdings)` | 보유 종목 목록                                      |      |
| 2-21 | `종목별 비중 알려줘`    | `get_portfolio_info(holdings)` | 종목별 비중 상위                                    |      |
| 2-22 | `삼성전자 보유 중이야?` | `get_portfolio_info(holdings)` | "네, 삼성전자 N주 보유 중이에요."                   |      |
| 2-23 | `LG전자 보유하고 있어?` | `get_portfolio_info(holdings)` | "LG전자는 현재 보유하고 있지 않아요." (미보유 종목) |      |

## 4. 대화 follow-up — 맥락 유지

> **순서대로 진행 필수**

### 4-A. 포트폴리오 → 종목 follow-up

| 순서 | 입력                          | 기대 응답                                                     | 결과 |
| ---- | ----------------------------- | ------------------------------------------------------------- | ---- |
| 1턴  | `포트폴리오 보여줘`           | 전체 템플릿                                                   |      |
| 2턴  | `제일 많이 내린 종목이 뭐야?` | worst_stock 종목명 + 수익률 (agent, get_portfolio_info(risk)) |      |
| 3턴  | `그 종목 현재가`              | 2턴에서 답한 종목의 현재가 카드                               |      |
| 4턴  | `그 종목 비중은?`             | 동일 종목 비중 (get_portfolio_info(holdings))                 |      |

### 4-B. 포트폴리오 → 대명사 follow-up

| 순서 | 입력                          | 기대 응답                                                                     | 결과 |
| ---- | ----------------------------- | ----------------------------------------------------------------------------- | ---- |
| 1턴  | `포트폴리오 보여줘`           | 전체 템플릿                                                                   |      |
| 2턴  | `제일 많이 오른 종목이 뭐야?` | best_stock 답변                                                               |      |
| 3턴  | `해당 종목 현재가`            | best_stock 현재가 카드 (이전에 "종목명을 입력해 주세요" 오류 발생했던 케이스) |      |
| 4턴  | `해당 종목 뉴스`              | best_stock 뉴스 카드                                                          |      |

### 4-C. 리스크 분석 → 수치 follow-up

| 순서 | 입력                           | 기대 응답                                                | 결과 |
| ---- | ------------------------------ | -------------------------------------------------------- | ---- |
| 1턴  | `리스크 지표 보여줘`           | risk 분석 + \_해설 기반 설명                             |      |
| 2턴  | `MDD가 어느 정도야?`           | 이전 tool 결과에서 MDD 수치 재참조 (새 tool 호출 불필요) |      |
| 3턴  | `회복하려면 얼마나 올라야 해?` | recovery_needed 수치 답변                                |      |

### 4-D. 수익률 → 비교 follow-up

| 순서 | 입력                                  | 기대 응답                                       | 결과 |
| ---- | ------------------------------------- | ----------------------------------------------- | ---- |
| 1턴  | `내 수익률 어때`                      | returns 분석                                    |      |
| 2턴  | `1개월이랑 6개월 중 어느 게 더 좋아?` | 두 수치 비교 (새 tool 호출 없이 이전 결과 참조) |      |
| 3턴  | `수익률 최저 종목은 뭐야?`            | worst_stock 답변                                |      |
| 4턴  | `그 종목 현재가 알려줘`               | worst_stock 현재가 카드                         |      |

### 4-E. 섹터 → 구성 follow-up

| 순서 | 입력                                      | 기대 응답                                 | 결과 |
| ---- | ----------------------------------------- | ----------------------------------------- | ---- |
| 1턴  | `섹터 비중 알려줘`                        | sector 분석                               |      |
| 2턴  | `국내랑 해외 비율이 어때?`                | domestic/foreign ratio (이전 결과 재참조) |      |
| 3턴  | `아까 본 거 기준으로 비중 1위 종목 뭐야?` | stock_concentration 1위 종목              |      |

## 6. 엣지 케이스 / 오류 방지

| #   | 입력                                   | 기대 응답                             | 확인 포인트                                                       | 결과 |
| --- | -------------------------------------- | ------------------------------------- | ----------------------------------------------------------------- | ---- |
| 6-1 | `수익률이 가장 높은 종목`              | best_stock 답변                       | "SK은 미보유" 오류 없어야 함 (\_PORTFOLIO_ANALYSIS_KW 가드)       |      |
| 6-2 | `포트폴리오에서 삼성전자 비중 얼마야?` | 삼성전자 비중 % 답변                  | 보유 종목이므로 미보유 안내 안 나와야 함                          |      |
| 6-3 | `포트폴리오에서 LG전자 비중 얼마야?`   | "LG전자는 현재 보유 종목이 아니에요." | 미보유 종목 → 미보유 안내                                         |      |
| 6-4 | `포트폴리오 내 손익`                   | risk 분석 (평가손익 포함)             | \_PORTFOLIO_KW_RE → dispatcher에서 chart_price 오라우팅 방지 확인 |      |
| 6-5 | `평가손익 알려줘`                      | unrealized_pnl 수치                   | get_stock_price가 아닌 get_portfolio_info 호출 확인               |      |
| 6-6 | `삼성전자 살까?`                       | 투자 추천 불가 안내                   | invest_advice 라우팅 (포트폴리오 경로 아님)                       |      |

## 7. \_해설 품질 체크

> agent가 `_해설` 필드를 제대로 활용하는지 응답 내용으로 확인

| #   | 입력                       | 기대 동작            | 확인 포인트                                       | 결과 |
| --- | -------------------------- | -------------------- | ------------------------------------------------- | ---- |
| 7-1 | `리스크 분석해줘`          | risk \_해설 활용     | "MDD", "변동성", "회복" 단어 응답에 포함되어야 함 |      |
| 7-2 | `내 수익률 분석해줘`       | returns \_해설 활용  | 기간별 수익률 + best/worst 종목 언급              |      |
| 7-3 | `포트폴리오 구성 분석해줘` | sector \_해설 활용   | 국내/해외 비중 + 상위 종목 언급                   |      |
| 7-4 | `거래 통계 분석해줘`       | stats \_해설 활용    | 승률·손익비 언급                                  |      |
| 7-5 | `보유 종목 구성 분석해줘`  | holdings \_해설 활용 | 보유 수 + 비중 1위 종목 언급                      |      |
